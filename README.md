# Solução Empresarial Istio em AKS com Terraform e Multi-Cluster

![Arquitetura da Solução](https://i.imgur.com/your-architecture-diagram.png)  <!-- Substituir com o link real do diagrama -->

## Visão Geral

Este projeto implementa uma arquitetura de microsserviços robusta, escalável e segura no **Azure Kubernetes Service (AKS)**, gerenciada por **Istio Service Mesh**. A infraestrutura como código (IaC) é totalmente automatizada com **Terraform**, e o ciclo de vida da aplicação é orquestrado via **GitHub Actions**, seguindo as melhores práticas de GitOps.

A solução foi projetada para cenários de alta demanda, suportando testes de carga de até **600.000 requisições por segundo (RPS)**, e oferece uma arquitetura multi-cluster para alta disponibilidade e resiliência. A integração com **Azure API Management (APIM)**, **Azure Cosmos DB** e um stack completo de **observabilidade** (Prometheus, Grafana, Jaeger) garante uma plataforma completa e pronta para produção.

## Arquitetura da Solução

A arquitetura é composta por três clusters AKS, cada um em uma VNet separada, interconectados via VNet Peering para formar uma malha de serviços multi-cluster com Istio.

### Componentes Principais:

- **Azure Kubernetes Service (AKS):** Orquestração de contêineres gerenciada.
- **Istio (Managed Add-on):** Malha de serviços para segurança, observabilidade e gerenciamento de tráfego.
- **Azure API Management (APIM):** Gateway de API para expor, proteger e gerenciar APIs.
- **Azure Cosmos DB:** Banco de dados NoSQL multi-modelo, distribuído globalmente.
- **Azure Key Vault:** Armazenamento seguro de segredos e certificados.
- **Azure Monitor:** Coleta e análise de métricas e logs (incluindo Prometheus gerenciado).
- **NGINX Ingress Controller:** Flexibilidade avançada para roteamento de entrada.
- **KEDA:** Autoscaling de pods baseado em eventos.
- **GitHub Actions:** Automação de CI/CD e fluxos de GitOps.

## Guia de Deployment

Para um guia detalhado, consulte o arquivo [DEPLOYMENT_GUIDE.md](./docs/DEPLOYMENT_GUIDE.md).

1.  **Configurar Autenticação:**
    - Faça login na CLI do Azure: `az login`
    - Exporte as credenciais do seu Service Principal como variáveis de ambiente:
      ```bash
      export ARM_CLIENT_ID="your-sp-client-id"
      export ARM_CLIENT_SECRET="your-sp-client-secret"
      export ARM_SUBSCRIPTION_ID="your-subscription-id"
      export ARM_TENANT_ID="your-tenant-id"
      ```

2.  **Inicializar o Terraform:**
    - Navegue até o diretório `terraform/`.
    - Execute `terraform init` para inicializar os providers e o backend.

3.  **Planejar e Aplicar:**
    - Crie um arquivo `terraform.tfvars` para customizar as variáveis (e.g., nomes de recursos, regiões).
    - Execute `terraform plan -out=tfplan` para revisar as mudanças.
    - Execute `terraform apply tfplan` para provisionar toda a infraestrutura.

O processo pode levar de 30 a 60 minutos, pois provisionará múltiplos clusters AKS e outros serviços do Azure.

## Observabilidade

A solução vem com um stack de observabilidade de ponta, acessível através dos seguintes endpoints (os hostnames são configuráveis):

- **Grafana:** `https://grafana.your-domain.com`
- **Jaeger:** `https://jaeger.your-domain.com`
- **Kiali:** `https://kiali.your-domain.com`
- **Alertmanager:** `https://alertmanager.your-domain.com`

Os dashboards pré-configurados no Grafana fornecem visibilidade completa sobre a saúde da malha Istio, performance das aplicações, métricas de infraestrutura e KPIs de negócio.
